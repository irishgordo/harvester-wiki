The maturity of a Harvester feature can be categorized into these stages:

### Experimental

  The feature is for testing purposes. It might cause data loss and the UX/API can change in the later releases.

### Technical Preview

  The feature is more stable and ready for public preview. UX/API is not supposed to change after entering GA.

### GA

  The feature is generally available. It's well-tested and incompatible API changes will come with an upgrade or migration plan.
